General notes, terminology and policies

  • There are two environments in which you can work;
    • The sandbox environment is where you can test and try things out to familiarize yourself with the steps you will be going through. The sandbox environment starts with dev. Our sandbox environment is https://dev.raleighhog.com
    • The production environment is the site that all members see. This is where you make the changes that chapter members see. Our production environment is https://www.raleighhog.com
  • Please note that prior to doing anything in the production environment you can (and should) test/practice in the sandbox environment first. This environment is used for testing and is not seen by members.
    • Your login credentials are the same for both environments
    • Your webmaster will help with any questions you may have
  1. There are two Event Calendars
    • Chapter Event Calender - the main calendar, anyone on our website can see all events
    • Road Captain Event Calendar - viewable only by Raleigh NC Chapter Road Captains after successful login to their account
    • Note: The calendar in the smartphone application (HogScan) is synced with the website Event calendar. Populating the website calendar populates the application's calendar.
  2. 'Event' is a ride, meeting or activity that is posted on the Raleigh NC Chapter Event Calendar. Throughout this tutorial they will all be referred to as event.
  3. Most of our events are 'closed', meaning members may invite a guest but the event can only take place if there is greater than 51% current Raleigh NC Chapter members. This is a HOG national policy and is due to insurance concerns.
    • LoH 'open to all' events are closed events. The 'open to all' statement means the event is not just for LoH members but is open to all Raleigh NC Chapter members, National HOG members and other guests as desired.
  4. A best practice is to not delete an event
    • If an event is cancelled or postponed due to inclimate weather, or some other reason, put **CANCELLED** or **POSTPONED** in front of the title and add a short message in the body of the event to keep members informed.
  5. Consistancy is key. Do not use different terminology for the same event. For example, throughout the website and in email campaigns our monthly meeting is referred to as "Monthly Chapter Meeting". It should be referred to as such in the Event Calendar so the naming convention is consistant throughout and transparent to end users.
    • It also helps when running year end reports. All "Monthly Chapter Meeting" can be grouped. For our Mar - Oct dinner rides a naming convention of "Dinner ride - xxx" (where xxx is the name of the restaurant) makes running year end reports easier.
  6. Last names, phone numbers and/or member personal email addresses are not to be displayed in any forward facing document, so they are not put in any event. The Raleigh NC Chapter Event Calendar is forward facing.
    • 'Forward facing' means any page that a non-authenticated user has access to. A non-authenticated user is one that has not successfully logged in.
    • The policy of maintaining privacy by not exposing last names, phone numbers, personal email addresses, personal motorcycle information, etc is from HOG national.
      • From the HOG Chapter Handbook, "Any content you list on your site shall not reveal private information about the members of your chapter, including names. If you need to use a name, use a nickname, first name and last initial or just a first name."
      • From the HOG Chapter Handbook, "Harley-Davidson discourages H.O.G.® chapters from publishing any photographs of minor children, even with permission from a parent or guardian". We will adhere to this policy.
  7. Please use caution when in the directories. You have the ability to delete files. This should never be done.
  • This tutorial covers creating an event and adding and image to a directory. You can click the respective link to jump to a specific section;
    • Create an Event
    • Add an Image to a Directory

Create an Event

  1. Login to your account on raleighhog.com and you will see a smaller Navigational Bar (navbar) on top of the Chapter navbar. See image below
  2. Click Add content to go to the following screen... See image below
    1. Article - creates a blog entry, not used here and a different tutorial
    2. Event - this is how Event Calender entries are made and is the purpose of this tutorial
    3. Event checkin - the check-in process uses the HOG[SCAN] app, this is not used
    4. Forum topic - forum has been turned off for the website, this is not used
    5. Gallery - you have access to add to the Gallery but this will be done by the Photographer, not used here and a different tutorial
  3. Click Event and the Create Event page opens
  4. There are four required fields. These fields, designated with a red asterisk, must be populated prior to saving the event.
    1. Title - A short, descriptive title
    2. EVENT DATE - Includes Date and Time
      1. Date - Event date formatted as mm/dd/yyyy
      2. Time - Event time formatted as hh:mm followed by am or pm
      3. Check the Show End Date box when the event is a multi-day event. The second Date / Time fields are populated with the last day of the event.
    3. Event type - our events (rides / meetings) are closed events
      1. Note: If Event type is Road Captain Events or RC pre-rides, the event will only appear on the Road Captain Event Calendar. (The RC Event Calendar can only be viewed by RC's after they've logged in.)
  5. Ride Mileage - this field is not used, it can be left blank. Fill it in if you know it as it may be incorporated in the future.
  6. Points - used only to identify newer members, leave at the default of 10
  7. Event Coordinator - select from the dropdown. If the name isn't in the list leave it blank
    1. There are members that have dual roles (officer & road captain). I've made it so that road captains have first name, last initial without a period. Select the coordinator from the road captains so if someone with a dual role changes roles, there will not be a change in the calendar.
  8. Image - leave blank. An image, if available, will be included in the Body area. (If populated here it messes with the copy flow and the map.)
  9. Body (Edit summary) - this is where the destination image (if available) and the event details are placed
    1. The image below shows the Body toolbar
      1. Hover over an icon to see a tooltip pop-up of the function of that icon. The icons outlined in red are ones that are of use. The rest can be ignored as there is no need to use HTML/CSS to style the copy.
        1. Link icon - e.g. used to provide a link to a hotel to make reservations for overnight trips
        2. Image icon - e.g. used to include an image of the destination in the copy. Images for frequent ride destinations, dealer events and meeting/parties have been preloaded and are currently available.
        3. Size dropdown - e.g. the font size defaults to 14px. It reads easier if you choose 16 from the dropdown.
        4. Bold icon - e.g. used to make text stand out. For instance, Login and click RSVP to repond
        5. Italic icon - e.g. the name of the restaurant or destination is usually in italics to make it stand out from the rest of the copy
    2. Click the image icon to add an image to the copy and the following pop-up appears. See image below
      1. Click the Browse Server button. The following window opens...
      2. Click the activities folder to expand/open it. It expands to show the following...
      3. Click the calendar_images folder to expand/open it. It expands to show the following...
      4. Click either dealer_events, destination-rides or meeting_or_party to expand/open to see the available images
      5. Select the image you wish to use. NOTE: single click the name to see a preview of the image. Double click the name to select it.
      6. You are brought back to the Image Properties pop-up and you can now see a preview of the image you chose.
      7. Tip: You do not want users to scroll through a large image to see the details of the event. In either the Width or Height boxes change the larger of the two to 400. The image aspect ratio will be maintained and users will appreciate seeing a smaller image.
      8. Click the OK button
      9. You can continue editing the Body to add text about the event.
        Tip: hit the spacebar so the text starts on the line below the image.
    3. Editing tip for the Body (Edit summary) section.
      1. Use the Shift+Enter keys to have a break in the line.
      2. Use the Enter key to start a new paragraph.
      3. Note that in the copy, the times listed match the format of the website timestamp (hh:mm). You want to keep things consistant.
      4. See image below for demonstraton
  10. EVENT LOCATION section
    1. Optional, does not have to be filled in
    2. If known, fill in Street, City & State
    3. Click the "Find Address on Map" button to populate the map
  11. Scroll to the bottom of the page
    1. Optionally fill in the 'Revision log message' section. Something as simple as "initial event create" is good when first creating the event so others looking can see what was done. If editing, something simple like "update event start time" is good enough. Again, so others looking can see what was done. (Members will not see these notes)
    2. Click the Save button to finish
  12. The event has been added to the appropriate calendar

Add an Image to a Directory

As time goes by, new ride destinations and events will be added to the calendar that may not have an image. The following steps will help you upload a new image to the appropriate directory.

This tutorial assumes you have the image you want to upload in a directory on your local machine.

As mentioned in the notes section at the top of this tutorial, use caution when in the directories. You have the ability to delete files. This should never be done.

  1. Login to your account on the website.
    1. In the navbar, click on My account.
    2. In the page that opens, click the File browser tab. see image below
  2. In the page that opens;
    1. Click on the activities subdirectory name to expand it
    2. Then click on the calendar_images subdirectory name to expand it
    3. You will see three subdirectories, dealer_events, destination_rides and meeting_or_party (these should look familiar, this is where you went to get the images you used when creating an event following the instructions in the previous section) see image below
  3. This following steps show how to add an image to the destination_rides rides subdirectory. Select the appropriate subdirectory when adding an image. The steps will be the same.
    1. Click the subdirectory name where you want to upload (add) a new image.
    2. Then, in the toolbar at the top, click Upload. see image below
  4. In the pop-up that opens, click the Choose File button. see image below
  5. There is no screenshot but a pop-up will open showing files on your machine. In the directory where you have the image, double click the image name to select it.
    1. After doing so, you will see the image filename in the box next to the Choose File button
    2. Click the Upload button. see image below
  6. You will see the image filename has been added to the existing images and a preview of the image at the bottom since it is selected.
    1. If you have another image to add, click the Choose File button and repeat the instructions from step 4 above.
    2. Click the x in the pop-up to close it. see image below

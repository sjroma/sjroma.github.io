CMS Initial Setup Notes

Assignable Roles

HOG[SCAN] uses "role based permissions" which restricts network access based on each members role within your HOG chapter. This allows you, the webmaster, to control what roles each officer and member has to the different levels of the system. For example, not every primary officer wants, or needs, the Primary Officer role.

Remember, you have a sandbox (dev) environment to work in. Make changes in that environment first.

Roles on this page are listed by permissions, highest to lowest.

Please note that the Webmaster, Primary Officer and Membership Officer roles have access to member information. Assign these roles with caution.

Webmaster
- Access to member information
- All permissions
- File browser directories (click 'My account' then the 'File browser' tab)
   read/write access to the shared, primary_officers, activities, road_captains and membership directories
Primary Officer
- Access to member information
- Permissions to add Article, Basic page, Event, Forum topic, Gallery, Newletter and Team Member content
   access to read/edit member information and approve Blog comments
   ability to do Chapter Administration as well as see all Chapter Reports
- Can see Check-ins and RSVPs in the Event Calendar
- File browser directories (click 'My account' then the 'File browser' tab)
   read/write access to the shared, primary_officers, activities, road_captains and membership directories
Membership Officer
- Access to member information
- Permissions to add Article, Forum topic and Gallery content
   access to read/edit member information
   ability to do Chapter Administration as well as see all Chapter Reports
- Can see Check-ins and RSVPs in the Event Calendar
- File browser directories (click 'My account' then the 'File browser' tab)
   read/write access to the shared and membership directories
Editor
- No access to member information
- Permissions to add Article, Basic Page, Event, Forum topic, Gallery, Newsletter and Team Member content.
   permission to make some changes to the look of the overall website
- Cannot see Check-ins and RSVPs in the Event Calendar
- File browser directories (click 'My account' then the 'File browser' tab)
   read/write access to the shared directory
Activities Officer
- No access to member information
- Permissions to add Article, Event, Forum topic and Gallery content
- Can see Check-ins and RSVPs in the Event Calendar
- File browser directories (click 'My account' then the 'File browser' tab)
   read/write access to the shared and activities directories
Photographer
- No access to member information
- Permissions to add Forum topic and Gallery content
- Cannot see Check-ins and RSVPs in the Event Calendar
- File browser directories (click 'My account' then the 'File browser' tab)
   read/write access to the shared directory
Road Captain
- No access to member information
- No permissions but can see the RC Calendar and RC Forms & Documents page
- Can see Check-ins and RSVPs in the Event Calendar
- File browser directories (click 'My account' then the 'File browser' tab)
   read/write access to the road_captains and shared directories

Directories

In addition to the aforementioned directories, each assignable role also has read/write permission to a users directory. This directory is unique to each person, even those that share the same role cannot see another's user directory.

Use the read/write access to directories to your advantage. Do not just dump files or images into a directory. Rather, create subdirectories as needed. For example, in the activities directory create a subdirectory to place images for destination rides and another subdirectory for images for dealer events. It's clear to you, and to others, what the different directories are for.

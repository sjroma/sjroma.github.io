HOG[SCAN] Wishlist

"Spiders are the only web developers happy to find bugs"

In no particular order, these are some fixes and enhancements that webmasters using HOG[SCAN] would like to see incorporated

Bugs

Global token for Chapter Handbook needs to be updated
closed issue 22 Dec 2022. The token has been updated
Global token for member release form needs to be updated
- The global token [hogscan:member_release_url] needs to be updated to point to the new form, shown on page 60, in the new 2022 Chapter Handbook. This token is used in the Join the Chapter page.
Unable to download/view the Chapter Charter or Chapter Handbook when clicking the respective links.
- On the About page, clicking the link to download the Chapter Charter or Chapter Handbook fails. End user's see "This site can't be reached" message. This is because the code repeats the https protocol for each link. Webmasters, until there is an official fix, edit the 'About Us' page. Change the 'Text format' to 'HTML Code' and scroll down to the Chapter Charter & Handbook section. The start of the section looks like this: <strong>Chapter Charter &amp; Handbook</strong> Within that section change the following line <a href="https://[hogscan:charter-url]" rel="nofollow" target="_blank"> to <a href="[hogscan:charter-url]" rel="nofollow" target="_blank"> Do the same with the link to the Chapter Handbook. That is, delete https:// Be sure to scroll down and Save your changes
The smartphone app exposes member information to Road Captains
- The smartphone app should not expose member's HOG ID or ICE information to Road Captains. The app is the only place where member's private info is exposed to the Road Captain role.
Fix the GPX mapping feature on the event page
- Seems to have stopped working around the same time the H-D Ride Planner altered how it gens a GPX file
Mailer link points to an out of date URL
- The HOG[SCAN]-->Mailer link in the admins navbar points to an old/outdated link to Elastic Email. The link needs to be updated to point to https://mailer.hogscan.com

Enhancements

Show member mileage for Ride 365
- Currently on the member's page one can only see mileage for 2018, 2019 and 2020 Members want to see mileage for 2021 as well as the running total for 2022 (the current year). Members always want to see the current year as well as possibly two or three years back.
Upgrade to use a better campaign emailer
- Webmasters are aware there will be an improvement in HOG[SCAN] V2.0 with an alternative to Elastic Email On the list because it's something everyone is anticipating
Leave user's on the same page when logging out rather than sending them to the home page each time
- Currently when a user logs out they are thrown back to the home page rather than leaving them on the page they were currently viewing. When viewing the calendar and logging out, stay on the calendar page. If viewing newsletters, leave them on the default newsletter page that says "You must be logged in to view our Newsletters." In other words, it should manage state.
Show Ride Mileage on the event page if the field is non-zero
- Currently the Ride Mileage is not shown in the copy of the event if it's been filled in. If the Ride Mileage field for an event has been filled in (i.e. it's not the default of zero) then show the mileage in the event copy. Much like an Event Coordinator is shown when that field has been filled in.
Give webmasters the ability to sync the sandbox environment with the production environment
- Currently a request has to be made to the Admin in order to have the sandbox environment synced up with the production environment. It would be nice if webmaster's had a button to click that would run the job to sync the sandbox with production so it could be done on-demand.

Webmasters, is there something you would like to see included in this list? Send me an email and I'll place it here.

Please, be descriptive. Are you reporting a bug or an enhancement? For a bug, what do you see versus what should you see? For an enhancement, what you would like to see that is currently not available?

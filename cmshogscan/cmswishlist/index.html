HOG[SCAN] Wishlist

In no particular order these are some enhancements and fixes that webmasters using HOG[SCAN] would like to see incorporated

Enhancements

Show member mileage for Ride 365
- Currently on the member's page one can only see mileage for 2018, 2019 and 2020 Members want to see mileage for 2021 as well as the running total for 2022 (the current year). Members always want to see the current year as well as possibly two or three years back.
Upgrade to use a better campaign emailer
- Webmasters are aware there will be an improvement in HOG[SCAN] V2.0 with an alternative to Elastic Email On the list because it's something everyone is anticipating
Leave user's on the same page when logging out rather than sending them to the home page each time
- Currently when a user logs out they are thrown back to the home page rather than leaving them on the page they were currently viewing. When viewing the calendar and logging out, stay on the calendar page. If viewing newsletters, leave them on the default newsletter page that says "You must be logged in to view our Newsletters." In other words, it should manage state.
Show Ride Milage on the event page if the field is non-zero
Currently the Ride Milage is not shown in the copy of the event if it's been filled in. If the Ride Milage field for an event has been filled in (i.e. it's not the default of zero) then show the milage in the event copy. Much like an Event Coordinator is shown when that field has been filled in.

Bugs

Unable to download/view the Chapter Charter or Chapter Handbook when clicking the respective links.
On the About page, clicking the link to download the Chapter Charter or Chapter Handbook fails. End user's see "This site can't be reached" message. This is because the code repeats the https protocol for each link.
Webmasters, until there is an official fix, edit the 'About Us' page. Change the 'Text format' to 'HTML Code' and scroll down to the Chapter Charter & Handbook section. The start of the section looks like this: <strong>Chapter Charter &amp; Handbook</strong>
Within that section change the following line
<a href="https://[hogscan:charter-url]" rel="nofollow" target="_blank">
to
<a href="[hogscan:charter-url]" rel="nofollow" target="_blank">
Do the same with the link to the Chapter Handbook. That is, delete https://
Be sure to scroll down and Save your changes
Fix the GPX mapping feature on the event page
- Seems to have stopped working around the same time the H-D Ride Planner altered how it gens a GPX file

Webmasters, is there something you would like to see included in this list? Send me an email and I'll place it here.

Please, be descriptive. Are you reporting an enhancement or a bug? For an enhancement, what you would like to see that is currently not available? For a bug, what do you see versus what should you see?
